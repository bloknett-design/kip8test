## Установка в репозиторий

1. Распакуйте архив в корень репозитория `kip8test`
2. Замените существующие файлы:
   - `index.html` → обновлённый (CSS + JS для таблиц и изображений)
   - `sw.js` → обновлённый (CACHE_VERSION = kipia-test-v38)
3. Добавьте новые/обновлённые JSON-файлы:
   - `data/pue/*.json` (9 файлов) + `data/pue/images/` (34 файла)
   - `data/pteep/3.6.json`, `data/pteep/meta.json` + `data/pteep/images/` (10 файлов)
   - `data/pote/*.json` (8 файлов)
   - `data/so153/*.json` (9 файлов) + `data/so153/images/` (12 файлов)
4. Закоммитьте: `git add . && git commit -m "v38: таблицы и изображения в документах + CSS-оптимизация"`
5. Запушьте: `git push`
6. Проверьте: https://bloknett-design.github.io/kip8test/

## Структура данных

### JSON-файлы глав (без изменений)
```json
{
  "chapter": "1.7",
  "title": "Заземление и защитные меры электробезопасности",
  "section": "1",
  "section_title": "Общие правила",
  "points": [
    { "num": "1.7.1", "title": "...", "text": "Текст пункта..." },
    { "num": "1.7.4", "title": "...", "text": "Текст\n<table class=\"doc-table\">...</table>", "has_html": true },
    { "num": "1.7.22", "title": "...", "text": "Текст\n<img src=\"data/pue/images/pue-053-005.jpg\" class=\"doc-image\">", "has_html": true }
  ]
}
```

### Поля
- `has_html: true` — пункт содержит HTML-таблицы или `<img>` теги (рендерится без экранирования)
- `<table class="doc-table">` — HTML-таблица (оборачивается в `.doc-table-wrap` для скролла)
- `<img src="data/{doc}/images/..." class="doc-image" loading="lazy">` — изображение (клик → полноэкранный просмотр)
